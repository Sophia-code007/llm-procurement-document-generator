# 采购文档智能生成系统 架构设计 v1.0

## 一、架构总览
本系统采用**前后端分离的分层架构**设计，自上而下划分为五层：用户前端层、API通信层、FastAPI后端服务层、数据存储层、外部依赖层。各层职责清晰、模块解耦，通过标准化接口进行交互，保障系统的可维护性与扩展性。

系统核心业务链路：用户在前端以自然语言输入采购需求 → 后端调用大模型完成语义解析与结构化提取 → 匹配对应采购文档模板并动态填充生成完整文档 → 自动执行合规风险检测 → 支持Word/PDF格式导出与项目全生命周期管理。

## 二、分层架构详细说明

### 1. 用户前端层
- **组成节点**：用户浏览器/PC端、Vue前端Web界面
- **技术载体**：Web单页应用，运行于用户本地浏览器环境
- **核心职责**：
  - 提供采购需求录入、参数可视化确认、文档在线预览、风险结果展示、历史项目管理等交互界面
  - 发起HTTP请求与后端服务进行数据交互
  - 处理页面渲染、表单校验、用户操作响应、文件下载触发

### 2. API通信层
- **组成节点**：HTTP / RESTful API 网关
- **请求方式**：支持 POST / GET 标准HTTP方法
- **核心职责**：
  - 承接前端所有请求，根据路由规则分发至后端对应服务模块
  - 统一处理参数校验、跨域配置、响应封装、异常拦截
  - 遵循RESTful设计规范，提供标准化接口能力，实现前后端解耦

### 3. FastAPI后端服务层
系统核心业务逻辑层，基于FastAPI框架搭建，包含5个独立功能模块，各模块内聚实现对应业务能力：

| 模块名称 | 核心职责 | 对应业务能力 |
|---------|---------|-------------|
| AI解析服务模块 | 调用大模型，实现自然语言到结构化JSON的转换 | 采购需求语义理解、字段提取、格式校验、结构化输出 |
| 模板生成服务模块 | 模板匹配、字段填充、文档组装 | 根据采购类别匹配对应模板，动态注入参数，组装生成完整采购文档 |
| 风险检测服务模块 | 合规校验、风险识别、条款审查 | 识别采购文档中的商务风险、合规漏洞、条款缺失，输出风险清单与优化建议 |
| 文件导出服务模块 | Word生成、PDF转换、下载 | 将生成的文档渲染为Word、PDF格式，提供文件下载能力 |
| 项目管理模块 | 需求录入、项目列表、版本管理 | 管理所有生成项目的元数据，支持历史查询、版本追溯、项目详情查看 |

### 4. 数据存储层
承载系统所有数据持久化能力，分为结构化数据存储与文件存储两部分：
- **MySQL数据库**
  - 存储内容：项目信息、用户数据、生成记录、参数快照等结构化数据
  - 交互关系：后端5个服务模块均通过读写操作与数据库交互，实现业务数据的持久化
- **文件存储**
  - 存储内容：生成完成的Word、PDF格式采购文档文件
  - 交互关系：由文件导出服务模块写入生成文件，由前端下载请求读取输出

### 5. 外部依赖层
- **组成节点**：大模型API服务（DeepSeek Open API）
- **核心作用**：为AI解析服务模块提供大语言模型能力支撑，实现自然语言理解、结构化字段提取等核心AI能力
- **交互关系**：AI解析服务模块通过API调用方式请求大模型服务，获取解析结果后返回业务层

## 三、核心业务主流程
结合架构流转，系统完整业务执行链路如下：
1. 用户在Vue前端界面输入自然语言采购需求，提交后发起HTTP请求
2. API通信层接收请求，经路由分发至AI解析服务模块
3. AI解析服务模块调用外部DeepSeek大模型API，将自然语言需求解析为标准化结构化JSON参数
4. 解析结果经API层返回前端，用户可对参数进行可视化确认与修改
5. 用户触发生成文档操作，请求分发至模板生成服务模块
6. 模板生成服务根据采购类别匹配对应模板，注入参数完成文档组装，同步将项目数据写入MySQL数据库
7. 风险检测服务模块读取生成的文档内容，执行合规风险识别，输出风险检测结果
8. 文件导出服务模块将最终文档渲染为Word/PDF格式，写入文件存储系统
9. 文档预览内容、风险检测结果、下载链接经API层返回至前端界面展示

## 六、架构设计原则
1. **分层解耦**：各层级职责边界清晰，层间通过标准化接口交互，单层修改不影响其他层级
2. **模块独立**：后端服务按业务域拆分，单个模块迭代、替换不影响整体系统
3. **可扩展性**：预留扩展能力，后续新增文档类型、风险规则、AI能力可在对应模块内平滑接入
4. **标准化**：统一接口规范、统一数据格式、统一命名规则，保障全链路字段与格式一致性
5. **可维护性**：结构清晰，职责明确，便于团队分工开发与后续运维迭代
