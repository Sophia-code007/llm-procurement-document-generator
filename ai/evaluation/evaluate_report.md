# AI 解析 Prompt 评估报告

## 评估对象

本次评估对象为 `ai/prompts/requirement_parse_prompt.md`，版本从 v1.0 调整为 v1.1。评估用例来自 `ai/evaluation/ai_parse_test_cases.csv`，共 20 条，覆盖数量提取、金额换算、单位识别、禁止编造和复杂要求遗漏五类场景。

## 评估结论

v1.1 的核心改动是统一枚举值、补充数量与单位字段、明确空值策略。相比 v1.0，v1.1 更适合后端 Schema 校验和 Apifox 接口联调，减少了“Prompt 输出中文枚举、Schema 要英文枚举”导致的解析失败。

| 指标 | v1.0 观察结果 | v1.1 预期结果 |
|---|---|---|
| 字段完整性 | 基本完整，但数量和单位未单独建字段 | 增加 `quantity`、`unit`，便于模板渲染 |
| 枚举一致性 | `purchaseCategory` 在 Prompt 中为中文，Schema 中为英文 | 统一使用英文枚举编码 |
| 空值策略 | 部分布尔字段容易默认 false | 未提及时返回 null，避免模型替用户判断 |
| 金额换算 | 已明确统一为元 | 保持统一为元 |
| 反编造能力 | 已强调未提及字段为 null | Few-shot 中强化极简输入示例 |

## 测试用例覆盖

| 类型 | 用例编号 | 检查重点 |
|---|---|---|
| 数量提错 | TC001-TC004 | 区分主采购数量与附属配置数量 |
| 金额提错 | TC005-TC008 | 区分单价、分项金额和总预算 |
| 单位提错 | TC009-TC012 | 正确识别米、平方米、套、人天等单位 |
| 凭空编造 | TC013-TC016 | 未提及交付、质保、安装时不得补充 |
| 遗漏要求 | TC017-TC020 | 复杂技术要求、服务内容、资质要求不可漏提 |

## 典型问题与修正

| 问题 | 影响 | 修正方式 |
|---|---|---|
| 中文/英文枚举混用 | 后端 Schema 校验失败 | 将 `purchaseCategory`、`purchaseMethod`、`paymentMode` 全部改为英文编码 |
| 布尔字段默认 false | 会把“不知道”误表达为“否” | 未提及时统一返回 null |
| 采购清单缺少数量和单位 | 模板难以生成“采购数量”相关段落 | 在 `purchaseItemList` 子项中补充 `quantity` 和 `unit` |
| 极简输入被过度补全 | 输出包含用户没说的交付、质保、联系人 | 在 Few-shot 示例中明确大量字段为 null |

## 后续测试方式

后端接入模型后，可按 `ai_parse_test_cases.csv` 逐条输入采购需求，将模型输出与“标准答案”列对比。初期建议人工检查每条用例的项目名称、数量、单位、预算金额、交付期限、质保期限和关键技术要求；稳定后再编写自动化字段校验脚本。

## 当前版本判断

v1.1 已满足课程项目 MVP 阶段的 AI 解析交付要求，可交付后端联调用。风险检测 Prompt 和摘要 Prompt 已具备规范框架，可作为后续增强能力接入。
